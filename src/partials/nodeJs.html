Databases:

  • DB divides into 2 main groups: db divide
    1. SQL (relatively): MySQL, SQL-Lite, PostgreSQL
    2. noSQL: MongoDB, Redis

  • MongoDB create a new cluster:

    • Cloud Provider & Region - live "AWS" and choose your nearest city
    • Press "Create cluster"
    • go "Database Access" & "ADD NEW DATABASE USER"
    • go "Network Access" & check from what IP-addresses you have access (if we need to allow connection from any ip-addresses don't make any rules or add this rule)
    • go to "Clusters" menu and press "COLLECTIONS"
    • Rule:

      collections names always noun in plural, f.e. users, products

    • Info: MongoDB NodeJS Driver (another name MongoClient) - package that helps connects code of NodeJS with database, another way to connect with database is f.e. express lib. and mongoose lib

Authentication/authorization:

  • bcrypt lib working with stable nodeJS version
  • bcryptjs lib working with current (unstable) nodeJS version
    • Token consist:
    1. headers - algorithm and way of token creation (by default using const headers = {"alg": "HS256", "typ": "JWT"})
    2. payload - part that is encrypted ("id" f.e.)
    3. SECRET_KEY - usual string with any text

      4. f.e const token = 'qwertyuiop.asdfghjkl.zxcvbnm', where "qwertyuiop" - headers encrypted, "asdfghjkl" - payload encrypted, "zxcvbnm" - SECRET_KEY encrypted

      method from jsonwebtoken decrypts token and pull decrypted payload
